💡개요

    이 프로젝트는 HTML5와 CSS3를 처음 섭렵하고 제작되었으며, 기존 노후된 사이트를 요구자와 컨텍을 통해 니즈에 맞춰 리뉴얼 되었습니다. 요구사항으로는 기존의 디자인이 노후되었고, 관리가 안되는 느낌이 강해 가시적이면서 깔끔하고 세련된 디자인으로 사용자들이 관리가 잘되고 센터의 이미지와 부합되는 홈페이지로 보였으면 좋겠다는 점이 있었습니다. 추후 JavaScript, jQuery를 습득 한 뒤, 플러그인과 ES6를 활용한 서브페이지 업데이트 , 로그인,회원가입 form 을 수정하여 사용자에게 다양한 서비스를 제공하기위해 업데이트 하였습니다.

📍사용 기술 스택

  • JavaScript
    • ES6문법을 활용하여 서브페이지 상품의 데이터베이스를 생성하여 쉽게 관리 할 수 있게 구현하였습니다.
    • 로그인 유효성 검사를 구현하였습니다.
    • 회원가입 시 비밀번호 일치여부 기능을 구현하였습니다.
  • jQuery
    • 기존에 구현하였던 css로 작성한 슬라이드는 무한반복이 불가능했으나 이를 극복하기 위해 slick-slide 플러그인을 사용하여 메인비쥬얼을 무한 슬라이드로 구현하였습니다.
    • scrollTo 플러그인으로 부드러운 스크롤 이동 효과를 적용시켰습니다.

🔗상세기능

  • 12칼럼을 활용한 그리드시스템에 의한 컨텐츠 제작
  • 메인비쥬얼을 보편적으로 사용되는 slick-plugin 실적용
  • 서브페이지에 ES6의 모듈을 활용하여 데이터베이스 구현 및 동적페이지 제작
  • 로그인 / 회원가입 form 작성
  • **lodash 사용하여 정렬하기 구현해서 추가하고 적기**
  • **리뷰 작성 및 삭제 기능 구현해서 추가하고 적기**
    메인페이지 바로가기

    모바일페이지 바로가기

    서브페이지 바로가기
  • 📌 서브페이지의 상품들을 HTML으로 작성하지 않고, JavaScript의 ES6 모듈을 활용하여 제작하였습니다. 데이터베이스를 만들어 추가 삭제 될 상품을 쉽게 관리 할 수 있고, 기존의 HTML로 작성하였을 때보다 간결하고 쉽게 만들 수 있었습니다. 해당 작업을 통해 데이터베이스에서 정보를 불러와 화면에 송출 시키는 것에 대해 이해 할 수 있게 되었습니다.
    로그인페이지 바로가기
  • 📌 로그인 플로우차트가 많이 미숙하지만, 어떤 방식으로 작동하는지 구조에 대해 깊이 생각하는 계기가 되었습니다. 역량을 키워 좀 더 완벽한 로그인페이지로 업데이트 할 계획입니다.
    회원가입 동의 바로가기
  • 📌 input:chekbox태그의 상태를 JavaScript를 통해 제어가능 하게 구현하였습니다.
    회원가입
  • 📌 첫번째 두번째 비밀번호 칸의 입력되는 값을 비교하여 그 결과를 알려줍니다.
    📝후기 및 개선사항

  • 이번 프로젝트를 통해 HTML과 CSS만으로 제어의 한계를 느꼈으며, 더 많은 기술을 배워 다양한 기능을 구현하고 싶은 생각을 가지게 되었습니다. 또 기획의 중요성도 알게 되었는데, 촘촘하고 탄탄한 기획을 바탕으로 제작을 해야 프로젝트 진행 중 불필요한 요소들을 줄 일 수 있다고 느꼈습니다. 작업 중간중간 미처 생각하지 못한 부분들이 발견될 때마다 난처했었기 때문입니다.


    프로젝트를 진행하면서 요구자와 소통을 하며 UI UX 적 측면을 떠나 요구자의 입장에서도 생각해 볼 수 있었습니다. 제작을 의뢰하는 입장의 가치관이 달라 사용자 입장을 대변하며 조율이 필요하다고 느꼈습니다. 몇가지 레퍼런스를 제시하며 제가 구현 할 수 있는 범위내에서 타협, 합의하에 최종 결과물을 완성하게 되었습니다. 초기에는 HTML과 CSS로만 구현했기 때문에 기능적으로 아쉬운 부분이 많았는데, JavaScript를 배우고 메인비쥬얼,서브페이지,로그인페이지를 업데이트 했습니다. 곧 디테일페이지의 상품리뷰 부분도 사용자가 작성하여 추가 할 수 있게 업데이트 할 예정입니다. 첫 프로젝트였던 만큼 애정이 가기 때문에 추후에는 실제로 운영해도 될 만큼 좋은 사이트로 만드는 것이 목표입니다.

  • 디자인
    • 메인페이지 모바일 레이아웃 구성
    • 메인페이지 모바일 시안 제작

    • Flex를 활용한 레이아웃 구성
  • JavaScript
    • matchMedia 함수를 활용한 디바이스에 맞춘 이벤트 적용
  • jQuery
    • 드롭다운 메뉴 구현

  • 처음 진행하는 팀프로젝트 였기 때문에 소통과 분담에 중점을 두고자 하였습니다. 원활한 소통과 자료통합,일정관리의 편의를 위해 노션(협업툴) 사용을 추진시켰습니다. 노션을 사용하면서 좀 더 체계적으로 프로젝트를 진행 할 수 있었고, 개개인이 맡은 작업물을 공유하며, 장소에 구애받지 않고 소통 할 수 있었습니다. 작업 분야를 기획(분석/기획/리뉴얼방향성), 디자인(아이디어스케치/시안) , 개발(기능) 으로 나누어 원하는 작업을 맡는 식으로 진행하였습니다.
